# Project Specification Templates

Templates for defining project scope, requirements, and deliverables.

## Available Templates

### product-requirements-document.md
Comprehensive PRD template for new products and major features.

**Use when:**
- Launching a new product
- Building a major feature
- Need stakeholder alignment
- Defining product strategy

**Sections:**
- Executive summary
- Background and context
- Goals and success metrics
- User research and personas
- Functional requirements
- Technical considerations
- Timeline and milestones

### feature-specification.md
Lightweight template for individual feature specifications.

**Use when:**
- Adding a specific feature
- Sprint planning
- Quick feature definition
- Enhancement proposals

**Sections:**
- Feature overview
- User stories
- Requirements
- Acceptance criteria
- Implementation notes

### project-brief.md
High-level project overview for initial planning.

**Use when:**
- Kicking off a project
- Seeking initial approval
- Communicating vision
- Early-stage planning

**Sections:**
- Project summary
- Objectives
- Scope
- Stakeholders
- High-level timeline
- Resources needed

## Usage

1. Copy the relevant template to your project
2. Fill in sections with project-specific details
3. Review with stakeholders
4. Use as reference throughout development
5. Update as the project evolves

## Tips

- Start with the project-brief for initial planning
- Expand to full PRD for major initiatives
- Use feature-specification for iterative development
- Keep specifications version controlled
- Link to related technical specifications
- Update specifications as requirements change
